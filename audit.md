## Подтверждено в реализации

### 1) Архитектура
- Тяжелая сеть/сканирование выполняются на backend, UI отвечает за управление и отбор.
- Решение корректно для ограничений браузерной платформы.

### 2) Backend scanner
- Один активный скан на инстанс (защита от параллельных запусков).
- Корректная обработка timeout-кодов завершения.
- Потоковый парсинг доменов устойчив к разрезанным chunk-строкам.
- Реализована явная валидация `addr` до запуска scanner:
  - принимаются `IPv4`, `IPv6`, `hostname`, `localhost`,
  - отклоняются URL со схемой/путем/query,
  - поддерживается `[IPv6]`,
  - нормализуется `FQDN.`.
- Добавлены endpoint’ы состояния:
  - `GET /api/domains/scan/status`
  - `GET /api/domains/scan/last-result`

### 3) Frontend UX
- Countdown и статус скана идут от backend-данных.
- После F5 UI подхватывает текущий скан и продолжает отображение.
- При `429` frontend подключается к активному запуску вместо ошибочного сценария.
- Реализованы предварительный список, удаление, импорт, экспорт, сохранение состояния.

### 4) Доменные правила
- Усилена нормализация и фильтрация входных доменов (ручной ввод и импорт).
- Устранены дубликаты без учета регистра.
- Практично обработаны wildcard/комментарии/обертки URL/порты.

### 5) DevOps и эксплуатация
- RealiTLScanner собирается в multi-stage (pin commit, reproducible).
- В runtime добавлены необходимые утилиты.
- Исправлен `504` для длинных сканов через увеличенные nginx timeout’ы `/api/`.
- Скрипты install/update поддерживают и Compose v2 (`docker compose`), и v1 (`docker-compose`).
